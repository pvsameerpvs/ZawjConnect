# ZawjConnect - Project Guidelines

## Expo
Read the exact versioned docs at https://docs.expo.dev/versions/v57.0.0/ before writing any code.

## Code Style
- Clean code 
- Single Responsibility: one file = one concern
- Max ~150 lines per file; split into components/hooks/utils
- Clean Architecture: presentation / domain / data layers
- Folder structure: `src/features/<feature>/` (colocate screens, components, hooks)

## Packages
- Use modern best-in-class packages only
- Prefer built-in Expo APIs over third-party alternatives
- No unused dependencies

## Git
- Never auto-commit
- Never auto-push
- Only commit/push when explicitly instructed
